## 🚀 快速開始

### 介面概覽

Rainbow Gantt V2 採用 **Retro-Futurism（復古未來主義）** 設計風格，主要由以下部分組成：

```
┌─────────────────────────────────────────────┐
│  [頂部標題欄]         [搜尋] [NEW_TASK] [⚙️] │
├─────────────────────────────────────────────┤
│                                             │
│            [主要內容區域]                    │
│                                             │
├─────────────────────────────────────────────┤
│  [ACTIVE] [COMPLETED] [POC] [CALENDAR] [KB] │ ← 底部導航島
└─────────────────────────────────────────────┘
```

### 五大功能頁面

點擊底部導航島可在五個頁面間切換：

| 圖標 | 名稱 | 功能 |
|------|------|------|
| 📊 | **ACTIVE_PROJECTS** | 進行中的專案（甘特圖/列表視圖） |
| ✅ | **COMPLETED_LOG** | 已完成專案記錄 |
| 🧪 | **POC_PROTOCOLS** | 概念驗證提案（POC/實驗性項目） |
| 📅 | **CALENDAR** | 雙人日曆視圖（Ja/Jo） |
| 📖 | **KNOWLEDGE_BASE** | 學習筆記與知識庫 |

## 📋 主要功能頁面

### 1️⃣ ACTIVE_PROJECTS（進行中專案）

**功能**：管理所有進行中的專案任務

#### 視圖切換

右上角可切換顯示模式：
- **GANTT**：甘特圖視圖（時間軸可視化）
- **LIST**：列表視圖（表格形式）

#### 人員篩選

點擊中央的人員標籤篩選任務：
- **ALL**：顯示所有任務
- **JANUS**：只顯示 Janus 的任務
- **JOSEPH**：只顯示 Joseph 的任務

#### 統計儀表板

頁面上方顯示：
- **圓餅圖**：按類別分布（AI驅能、品牌行銷、客戶開發等）
- **總體進度**：完成百分比
- **任務統計**：各類別的進度條和數量

#### 甘特圖操作

**時間軸導航**：
- `< PREV`：上個月
- `NEXT >`：下個月
- `TODAY`：跳轉到當前月份

**任務條功能**：
- 不同顏色代表不同類別
- 任務條上顯示進度百分比
- 點擊任務條可編輯詳細資訊

**優先級標記**：
- 🔴 `URGENT`：紅色框（緊急）
- 🟠 `HIGH`：橙色框（高優先級）
- 🟡 `MEDIUM`：黃色框（中等）
- ⚪ `LOW`：灰色框（低優先級）

**負責人標籤**：
- 黑色方塊顯示負責人縮寫（JA/JO）

---

### 2️⃣ COMPLETED_LOG（已完成專案）

**功能**：查看和管理已完成的專案記錄

#### 顯示資訊

表格顯示以下欄位：
- **PROTOCOL_NAME**：專案名稱
- **AGENT**：負責人（頭像 + 姓名）
- **CATEGORY**：類別標籤（帶顏色）
- **START_DATE**：開始日期
- **END_DATE**：完成日期
- **DURATION**：執行天數（自動計算）
- **ACTIONS**：刪除按鈕

#### 操作

- **人員篩選**：同 Active Projects，可按人員篩選
- **添加完成項目**：點擊 `+ ADD_ENTRY` 添加已完成項目
- **刪除**：點擊紅色垃圾桶圖標刪除記錄

---

### 3️⃣ POC_PROTOCOLS（概念驗證）

**功能**：管理實驗性專案和技術驗證提案

#### 卡片顯示

每個 POC 以卡片形式顯示：
- **標題**：專案名稱
- **狀態標籤**：PENDING（待處理）、IN_PROGRESS（進行中）
- **類別標籤**：專案分類
- **負責人**：頭像 + 姓名
- **利益相關者**：團隊成員頭像（顯示前 3 個）
- **描述**：專案詳細說明

#### 操作

- **新增 POC**：點擊 `+ NEW_PROTOCOL`
- **編輯 POC**：點擊卡片任意位置
- **刪除 POC**：點擊右上角紅色垃圾桶

#### 空狀態提示

首次使用時顯示：
```
🧪 NO_PROTOCOL_FOUND
INITIATE_NEW_PROTOCOL_SEQUENCE_BELOW
[+ INITIATE]
```

---

### 4️⃣ CALENDAR（雙人日曆）

**功能**：Janus 和 Joseph 各自的工作日曆

#### 雙視角切換

左上角切換按鈕：
- **JA**（Janus）：粉紅色高亮
- **JO**（Joseph）：白色邊框

#### 月份導航

右上角：
- `<`：上個月
- `>`：下個月
- 中央顯示：**2025年12月**

#### 日曆網格

- **7×5 格式**：週日到週六
- **日期格子**：顯示當日所有內容
- **完整顯示**：不截斷文字（可滾動）

#### 添加/編輯內容

**點擊日期格子**：
- 進入全屏編輯模式
- 隱藏日曆網格
- 顯示編輯面板

**編輯面板功能**：
- **📝 內容輸入**：多行文字編輯
- **📎 附件上傳**：支持圖片（點擊預覽）
- **💾 保存**：綠色 SAVE 按鈕
- **❌ 取消**：灰色 CANCEL 按鈕

#### 圖片查看

點擊日曆內的圖片：
- 全屏 Lightbox 顯示
- 點擊背景或 X 關閉

---

### 5️⃣ KNOWLEDGE_BASE（知識庫）

**功能**：技術學習筆記和知識管理

#### 筆記卡片

每張筆記卡片顯示：
- **標題**：筆記主題
- **內容預覽**：Markdown 渲染
- **相關專案**：關聯的任務標籤
- **最後修改時間**：時間戳

#### 操作

- **新增筆記**：點擊 `+ NEW_ENTRY`
- **編輯筆記**：點擊卡片上的 📄 圖標
- **刪除筆記**：點擊 🗑️ 圖標

#### 編輯模式

**基本資訊**：
- **標題**：必填
- **內容**：支持 Markdown 格式
  - 標題：`# H1`, `## H2`, `### H3`
  - 列表：`- 項目` 或 `1. 項目`
  - 代碼塊：` ```語言\n代碼\n``` `
  - 粗體：`**文字**`

**附件管理**：
- **類型**：Link（連結）、File（文件）、Image（圖片）
- **名稱**：必填
- **URL**：連結地址（Link/Image）
- **備註**：可選
- **操作**：添加（+）/ 刪除（🗑️）

**關聯專案**：
- 點擊 `+ Add Related Protocol` 展開專案選擇器
- 勾選相關專案（可多選）
- 顯示為帶有 ❌ 的標籤（點擊移除關聯）

## 📝 任務管理

### 新增任務

#### 方式一：從 Active Projects 頁面

1. 點擊右上角 `+ NEW_TASK` 按鈕
2. 右側彈出編輯面板

#### 方式二：從 POC/Completed 頁面

1. 點擊對應頁面的 `+ NEW_PROTOCOL` 或 `+ ADD_ENTRY`
2. 系統自動設置對應類型

### 任務欄位說明

#### 基本資訊

| 欄位 | 說明 | 必填 |
|------|------|------|
| **任務名稱** | 專案標題 | ✅ |
| **負責人** | 選擇 Janus 或 Joseph | ✅ |
| **狀態** | Pending / In Progress / Completed / Blocked | ✅ |
| **優先級** | Urgent / High / Medium / Low | ❌ |
| **類別** | AI驅能 / 品牌行銷 / 客戶開發 / 流程優化 / 產品行銷 / 其他 | ✅ |

#### 時間設定

| 欄位 | 說明 | 必填 |
|------|------|------|
| **開始日期** | 專案開始日（格式：YYYY-MM-DD） | ✅ |
| **結束日期** | 專案結束日 | ✅ |
| **進度** | 0-100%（滑桿調整） | ❌ |

#### POC 專屬欄位

| 欄位 | 說明 | 必填 |
|------|------|------|
| **是否為 POC** | 勾選框（勾選後任務顯示在 POC_PROTOCOLS） | ❌ |
| **利益相關者** | 添加團隊成員（姓名 + 角色） | ❌ |
| **描述** | 詳細說明（支持多行） | ❌ |

#### 附件與資源

- **類型**：Link / File / Image
- **操作**：上傳圖片 / 添加連結 / 附加文件
- **預覽**：圖片可點擊放大查看

### 編輯任務

1. **從甘特圖**：點擊任務條
2. **從列表視圖**：點擊任務行
3. **從 POC 頁面**：點擊卡片

編輯面板右側彈出，修改後點擊 **SAVE** 保存。

### 刪除任務

1. 打開任務編輯面板
2. 滾動到底部
3. 點擊紅色 **DELETE** 按鈕
4. 確認刪除（無法復原）

## 👥 人員管理

### 打開設定面板

點擊右上角 **⚙️ Settings** 按鈕

### People Management（人員管理）

#### 功能

- **查看所有人員**：系統中的所有負責人
- **新增人員**：
  1. 在輸入框輸入名字
  2. 點擊綠色 `+ ADD` 按鈕
- **編輯人員名稱**：
  1. 點擊人員卡片上的 ✏️ 圖標
  2. 修改名稱
  3. 點擊 ✓ 保存或 ✗ 取消
  4. **自動同步**：所有關聯任務的 owner 欄位會自動更新
- **刪除人員**：
  1. 點擊紅色 🗑️ 圖標
  2. 確認刪除
  3. ⚠️ **注意**：關聯任務的 owner 會變成空白

#### 人員卡片顯示

```
┌─────────────────────┐
│ 👤 Janus          ✏️ │
│                      │
│ [✓ Save] [✗ Cancel] │ ← 編輯模式
│                   🗑️ │
└─────────────────────┘
```

### 關閉設定面板

點擊：
- 右上角 **✗** 按鈕
- 面板外的灰色遮罩區域

## 📅 日曆視圖

### 快速操作

#### 新增內容

1. 切換到對應人員（JA 或 JO）
2. 點擊目標日期
3. 在編輯器中輸入內容
4. 點擊 **SAVE**

#### 編輯內容

1. 點擊已有內容的日期
2. 修改文字或上傳圖片
3. 點擊 **SAVE**

#### 刪除內容

1. 點擊日期進入編輯模式
2. 清空所有內容
3. 點擊 **SAVE**（空內容會刪除該日記錄）

### 圖片管理

#### 上傳圖片

1. 點擊 **UPLOAD IMAGE** 按鈕
2. 選擇圖片文件（支持所有常見圖片格式）
3. 圖片自動添加到附件列表
4. 保存後顯示在日曆格子中

#### 查看圖片

- **日曆格子內**：點擊縮圖
- **編輯模式**：點擊附件列表中的預覽圖
- **Lightbox 模式**：全屏查看（點擊背景或 X 關閉）

#### 刪除圖片

1. 進入編輯模式
2. 點擊圖片右上角的 **✗** 按鈕
3. 保存變更

### 最佳實踐

#### 任務命名

✅ **好的命名**：
- `平台官網上線`（具體、可量化）
- `API 整合測試`（清晰的技術目標）
- `客戶需求訪談 - ABC 公司`（包含對象）

❌ **避免的命名**：
- `做事情`（太模糊）
- `任務1`（無意義）
- `修bug`（缺乏細節）

#### 進度更新

建議每週更新一次任務進度：
1. 打開 Active Projects 頁面
2. 點擊任務進入編輯
3. 調整進度滑桿（0-100%）
4. 保存

#### POC 使用場景

適合放在 POC_PROTOCOLS 的專案：
- ✅ 技術驗證（新框架、新工具）
- ✅ 概念測試（MVP、原型）
- ✅ 實驗性功能
- ✅ 短期研究項目

不適合：
- ❌ 已確定的正式專案（應放 Active Projects）
- ❌ 已完成的項目（應放 Completed Log）

#### 日曆使用建議

每個人的日曆可用於記錄工作日誌、會議紀要、決策記錄或其他需要追蹤的內容。

## 🔧 疑難排解

### 常見問題

#### Q: 任務不顯示在甘特圖上？

**A:** 檢查以下項目：
1. 任務是否勾選「是否為 POC」（POC 任務不顯示在甘特圖）
2. 任務日期是否在當前月份範圍內（使用 PREV/NEXT 切換月份）
3. 任務是否設置了開始和結束日期

#### Q: 人員篩選後看不到任務？

**A:**
- 確認任務的 owner 欄位是否設置
- 確認 owner 名稱拼寫是否與篩選條件一致（區分大小寫）

#### Q: 日曆內容保存後消失？

**A:**
- 確認點擊了 **SAVE** 按鈕而非 **CANCEL**
- 檢查切換的是否為正確的人員（JA/JO）
- 刷新頁面重新載入

#### Q: 圖片上傳失敗？

**A:**
- 確認圖片大小不超過 5MB（系統會提示）
- 確認圖片格式為常見圖片格式
- 檢查網路連線

#### Q: Settings 面板無法打開？

**A:**
- 嘗試刷新頁面
- 確認沒有其他彈窗阻擋（關閉任務編輯面板）

#### Q: 甘特圖任務條顯示不完整？

**A:**
- 任務跨多個月時，只顯示當前月份的部分
- 使用 PREV/NEXT 切換月份查看完整任務

## 🎨 設計理念

Rainbow Gantt V2 採用 **White Cassette Futurism（白色磁帶未來主義）** 設計語言：

- **Retro Computing**：復古電腦終端美學
- **Monospace Typography**：等寬字體，呼應早期編程環境
- **Minimalist Colors**：極簡配色，突出資訊層次
- **Geometric Borders**：硬朗的幾何邊框
- **Flat Shadows**：扁平化陰影效果

這種設計風格結合了 80 年代電腦美學與現代 UI/UX 實踐，創造出獨特的視覺體驗。
